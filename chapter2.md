## 一、信息存储基础

### 字节与内存

- 计算机以**字节（8位）**为最小寻址单位
- 每个字节有唯一的**内存地址**
- 所有数据（整数、浮点、指针、程序）都存储为字节序列

### 字长（Word Size）

- 字长决定**虚拟地址空间**的大小
- 32位字长：地址空间 $2^{32}$ 字节 ≈ 4GB
- 64位字长：地址空间 $2^{64}$ 字节 ≈ $1.8 \times 10^{19}$ 字节

### C语言数据类型大小（64位Linux）

| 类型 | 大小（字节） |
|------|------------|
| `char` | 1 |
| `short` | 2 |
| `int` | 4 |
| `long` | 8 |
| `long long` | 8 |
| `char *`（指针） | 8 |
| `float` | 4 |
| `double` | 8 |

> 注意：Windows 64位下 `long` 为4字节（LLP64模型），Linux/macOS 为8字节（LP64模型）

### 字节顺序（Endianness）

**大端序（Big Endian）**：最高有效字节存储在最低地址

**小端序（Little Endian）**：最低有效字节存储在最低地址（x86-64采用）

示例：整数 `0x01234567` 存储在地址 `0x100`：

| 地址 | 大端序 | 小端序 |
|------|--------|--------|
| 0x100 | 01 | 67 |
| 0x101 | 23 | 45 |
| 0x102 | 45 | 23 |
| 0x103 | 67 | 01 |

字节序的影响：
- 网络通信（网络字节序为大端）
- 读取二进制文件
- 调试时查看内存转储

### 布尔代数与位运算

| 运算 | 符号 | 说明 |
|------|------|------|
| AND | `&` | 两者均为1才为1 |
| OR | `\|` | 至少一个为1则为1 |
| NOT | `~` | 取反 |
| XOR | `^` | 相同为0，不同为1 |

**位运算 vs 逻辑运算（C语言）**：

```c
// 位运算：对每一位操作
0x69 & 0x55  = 0x41
0x69 | 0x55  = 0x7D
~0x41        = 0xBE

// 逻辑运算：只看0/非0
0x69 && 0x55  = 1   // 两者均非零
0x00 || 0x55  = 1   // 至少一个非零
!0x41         = 0   // 非零取反为0
```

### 移位运算

| 运算 | 符号 | 高位填充 |
|------|------|---------|
| 逻辑左移 | `<<` | 低位补0 |
| 逻辑右移 | `>>` | 高位补0 |
| 算术右移 | `>>` | 高位补符号位 |

> C语言中，无符号数右移为逻辑右移，有符号数右移**几乎所有编译器**实现为算术右移（但技术上是未定义行为）

> 移位量必须满足 $0 \leq k < w$，否则行为未定义

## 二、整数表示

### 无符号数（Unsigned）

对于一个 $w$ 位的二进制数 $\vec{x} = [x_{w-1}, x_{w-2}, \ldots, x_0]$：

$$B2U_w(\vec{x}) = \sum_{i=0}^{w-1} x_i \cdot 2^i$$

- 最小值 $UMin_w = 0$
- 最大值 $UMax_w = 2^w - 1$
- **B2U是双射**（一一对应）

### 补码（Two's Complement）

$$B2T_w(\vec{x}) = -x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2} x_i \cdot 2^i$$

- 最高位为**符号位**，权重为 $-2^{w-1}$
- 最小值 $TMin_w = -2^{w-1}$（位模式：$10\ldots0$）
- 最大值 $TMax_w = 2^{w-1} - 1$（位模式：$01\ldots1$）
- **B2T也是双射**

### 其他整数表示（了解）

**原码（Sign-Magnitude）**：
$$B2S_w(\vec{x}) = (-1)^{x_{w-1}} \cdot \sum_{i=0}^{w-2} x_i \cdot 2^i$$

**反码（Ones' Complement）**：
$$B2O_w(\vec{x}) = -x_{w-1} \cdot (2^{w-1}-1) + \sum_{i=0}^{w-2} x_i \cdot 2^i$$

原码和反码均有 $+0$ 和 $-0$ 两种表示，现代计算机均使用补码。

### 常见字长的范围

| 字长 | UMin | UMax | TMin | TMax |
|------|------|------|------|------|
| 8位 | 0 | 255 | -128 | 127 |
| 16位 | 0 | 65535 | -32768 | 32767 |
| 32位 | 0 | $2^{32}-1$ | $-2^{31}$ | $2^{31}-1$ |
| 64位 | 0 | $2^{64}-1$ | $-2^{63}$ | $2^{63}-1$ |

### 重要性质

- $|TMin| = TMax + 1$（不对称性，最小值无对应正数）
- $UMax_w = 2 \times TMax_w + 1$
- $-1$ 的补码表示为全1：$11\ldots1$
- C语言中：`<limits.h>` 定义 `INT_MAX`, `INT_MIN`, `UINT_MAX` 等

## 三、有符号数与无符号数的转换

### 补码转无符号数（T2U）

$$T2U_w(x) = \begin{cases} x + 2^w & x < 0 \\ x & x \geq 0 \end{cases}$$

### 无符号数转补码（U2T）

$$U2T_w(u) = \begin{cases} u & u \leq TMax_w \\ u - 2^w & u > TMax_w \end{cases}$$

### 关键规律

- **位模式不变**，只是解释方式改变
- 非负数 ↔ 转换结果不变
- 负的补码数 → 转换为无符号数后变成大正数
- 大于 TMax 的无符号数 → 转换为补码后变成负数

### C语言中的隐式转换规则

**规则：** 有符号数和无符号数混合运算时，有符号数**隐式转换**为无符号数。

```c
// 表达式             类型      值
0 == 0U              无符号    1（true）
-1 < 0               有符号    1（true）
-1 < 0U              无符号    0（false！-1转为UMax）
2147483647 > -2147483648  有符号  1（true）
2147483647U > -2147483648 无符号  0（false！）
2147483647 > (int)2147483648U  有符号  1（true）
```

## 四、整数的扩展与截断

### 零扩展（Zero Extension）

用于**无符号数**从小类型转换到大类型，高位补0：

$$[0, \ldots, 0, x_{w-1}, \ldots, x_0]$$

值不变：$B2U_{w'}(\vec{x'}) = B2U_w(\vec{x})$

### 符号扩展（Sign Extension）

用于**有符号数（补码）**从小类型转换到大类型，高位补符号位：

$$[x_{w-1}, \ldots, x_{w-1}, x_{w-1}, x_{w-2}, \ldots, x_0]$$

值不变：$B2T_{w'}(\vec{x'}) = B2T_w(\vec{x})$

示例：
```
short -12345 = 1100 1111 1100 0111
int   -12345 = 1111 1111 1111 1111 1100 1111 1100 0111
（高位全部填充符号位1）
```

### 截断（Truncation）

丢弃高位，只保留低 $k$ 位：

- **无符号数截断**：$B2U_k([x_{k-1}, \ldots, x_0]) = B2U_w(\vec{x}) \bmod 2^k$
- **有符号数截断**：先按无符号截断，再用 B2T 解释

### 同时涉及大小和符号变化

**原则：先改变大小，再改变符号**（C语言标准规定）：

```c
char c = -1;             // 0xFF，值为-1
unsigned int u = c;
// 步骤1：char→int（符号扩展）：-1
// 步骤2：int→unsigned int：4294967295
// 结果：4294967295，不是255！

// 对比：
unsigned char uc = (unsigned char)c;  // 先转unsigned char：255
unsigned int u2 = uc;                 // 再零扩展：255
```

## 五、无符号数运算

### 无符号加法

$$x +_w^u y = (x + y) \bmod 2^w$$

展开为：
$$x +_w^u y = \begin{cases} x + y & x + y < 2^w \quad \text{（正常）} \\ x + y - 2^w & x + y \geq 2^w \quad \text{（溢出）} \end{cases}$$

**检测无符号加法溢出**：
$$s = x +_w^u y \text{ 溢出} \iff s < x \text{（或 } s < y\text{）}$$

### 无符号取反

$$-_w^u x = \begin{cases} 0 & x = 0 \\ 2^w - x & x > 0 \end{cases}$$

### 无符号乘法

$$x *_w^u y = (x \cdot y) \bmod 2^w$$

结果只保留低 $w$ 位，高位丢弃。

## 六、补码运算

### 补码加法

$$x +_w^t y = \begin{cases} x + y - 2^w & x + y \geq 2^{w-1} \quad \text{（正溢出）} \\ x + y & -2^{w-1} \leq x + y < 2^{w-1} \quad \text{（正常）} \\ x + y + 2^w & x + y < -2^{w-1} \quad \text{（负溢出）} \end{cases}$$

**检测补码加法溢出**：
- 正溢出：$x > 0,\ y > 0$，但 $s \leq 0$
- 负溢出：$x < 0,\ y < 0$，但 $s \geq 0$

### 补码取反

$$-_w^t x = \begin{cases} TMin_w & x = TMin_w \\ -x & x > TMin_w \end{cases}$$

注意：$TMin$ 的补码取反结果仍是 $TMin$（无对应正数，溢出）。

位级实现：$-x = \sim x + 1$（取反加一）

### 补码乘法

$$x *_w^t y = U2T_w\left((x \cdot y) \bmod 2^w\right)$$

**重要性质**：无符号乘法和补码乘法的**低 $w$ 位结果相同**：

$$T2U_w(x *_w^t y) = x *_w^u y$$

这就是为什么 CPU 只需要一套乘法器，无符号/有符号共用。

## 七、乘以常数的优化

编译器常用**移位和加减法**替代乘法（整数乘法指令延迟高达3~10个时钟周期）：

### 乘以2的幂

$$x \cdot 2^k = x \ll k$$

对有符号和无符号数均成立（忽略溢出）。

### 乘以任意常数

将常数表示为2的幂之和或差：

```c
x * 14:
  形式1（和）: x*(8+4+2) = (x<<3)+(x<<2)+(x<<1)  // 3次移位+2次加法
  形式2（差）: x*(16-2)  = (x<<4)-(x<<1)          // 2次移位+1次减法（更优）

x * 7:
  = x*(8-1) = (x<<3) - x
```

编译器会选择操作数最少的形式。

## 八、除以2的幂

### 无符号数右移（逻辑右移）

$$\lfloor x / 2^k \rfloor = x \gg_l k$$

高位补0，结果向下取整（等价于向零取整，因无符号数非负）。

### 有符号数右移（算术右移）

$$\lfloor x / 2^k \rfloor = x \gg_a k$$

高位补符号位，结果**向下取整**（向负无穷方向）。

**问题**：C语言要求整数除法**向零取整**，负数时两者不同：

```
-7 / 2 = -3（向零取整，C语言要求）
-7 >> 1 = -4（算术右移，向下取整）
```

### 修正偏置（使负数除法向零取整）

$$\lceil x / 2^k \rceil = (x + 2^k - 1) \gg_a k$$

汇编实现示例（`x / 4`，`x` 为有符号数在 `%rdi` 中）：

```asm
movq  %rdi, %rax
sarq  $63,  %rax     ; 全符号位：x≥0则%rax=0，x<0则%rax=-1（全1）
andl  $3,   %eax     ; 偏置量：x≥0则0，x<0则3（即 2^k-1）
addq  %rdi, %rax     ; x + 偏置
sarq  $2,   %rax     ; 算术右移2位
```

## 九、浮点数表示

### IEEE 754 标准格式

浮点数由三部分组成：$V = (-1)^s \times M \times 2^E$

| 字段 | 单精度（32位） | 双精度（64位） |
|------|-------------|-------------|
| 符号位 s | 1位 | 1位 |
| 阶码 exp | 8位 | 11位 |
| 尾数 frac | 23位 | 52位 |

### 三种情况

**1. 规格化值（Normalized）**：$exp \neq 00\ldots0$ 且 $exp \neq 11\ldots1$

- 阶码：$E = e - Bias$，其中 $e$ 为阶码字段的无符号值，$Bias = 2^{k-1} - 1$（单精度127，双精度1023）
- 尾数：$M = 1 + f$（隐含的前导1）
- 单精度：$E \in [-126, 127]$

**2. 非规格化值（Denormalized）**：$exp = 00\ldots0$

- 阶码：$E = 1 - Bias$（而不是 $0 - Bias$，目的是平滑过渡）
- 尾数：$M = f$（无隐含前导1）
- 用途：表示0（$f=0$）和非常接近0的数

**3. 特殊值**：$exp = 11\ldots1$

| frac | 值 |
|------|-----|
| $= 0$ | $\pm\infty$ |
| $\neq 0$ | NaN（Not a Number） |

### 数值示例（8位浮点：1位符号，4位阶码，3位尾数，Bias=7）

| exp | frac | 值 |
|-----|------|-----|
| 0000 | 000 | 0 |
| 0000 | 001 | $\frac{1}{512}$ |
| 0111 | 000 | 1.0 |
| 0111 | 100 | 1.5 |
| 1110 | 111 | 248 |
| 1111 | 000 | $\infty$ |
| 1111 | 001 | NaN |

### 浮点运算性质

- **不满足结合律**：$(3.14 + 1e10) - 1e10 \neq 3.14 + (1e10 - 1e10)$
- **不满足分配律**：$1e20 * (1e20 - 1e20) \neq 1e20*1e20 - 1e20*1e20$
- 满足**单调性**：$a \geq b \Rightarrow a+c \geq b+c$（忽略NaN）
- 浮点加法满足**交换律**但不满足结合律

### 舍入模式

| 模式 | 说明 |
|------|------|
| 向偶数舍入（默认） | 中间值舍入到最近偶数 |
| 向零舍入 | 截断小数部分 |
| 向下舍入 | 向负无穷方向 |
| 向上舍入 | 向正无穷方向 |

示例（向偶数舍入到整数）：

```
2.5 → 2（向偶数）
3.5 → 4（向偶数）
2.4 → 2（正常四舍五入）
2.6 → 3（正常四舍五入）
```

### 浮点数与整数转换

```c
int x = 1234567890;
float f = (float)x;   // 精度损失！float只有23位尾数
double d = (double)x; // 精确，double有52位尾数

// 浮点转整数：向零截断
(int)3.7  = 3
(int)-3.7 = -3
(int)1e10 = 未定义行为！（超出int范围）
```

## 十、位级表示的重要性质总结

### 加法的位级等价性

无符号加法和补码加法结果的**位表示相同**：
$$T2U_w(x +_w^t y) = T2U_w(x) +_w^u T2U_w(y)$$

### 乘法的位级等价性

无符号乘法和补码乘法结果的**低 $w$ 位相同**：
$$T2U_w(x *_w^t y) = T2U_w(x) *_w^u T2U_w(y)$$

这两条性质说明 CPU 可以用同一套硬件处理有符号/无符号运算。

## 十一、C语言中的常见陷阱

### 陷阱1：无符号数下溢导致死循环

```c
// 危险！
unsigned i;
for (i = n-1; i >= 0; i--)  // i永远>=0，死循环

// 安全写法
for (i = n; i > 0; i--)
// 或
for (int i = n-1; i >= 0; i--)  // 用有符号数
```

### 陷阱2：有符号/无符号混合比较

```c
int a = -1;
unsigned b = 1;
if (a < b)  // false！a被隐式转为UMax=4294967295
    printf("a < b");
else
    printf("a >= b");  // 实际输出这个
```

### 陷阱3：TMin 的特殊性

```c
int x = INT_MIN;   // -2147483648
int y = -x;        // 仍是 -2147483648！（溢出）
int z = abs(x);    // 未定义行为！
```

### 陷阱4：sizeof 返回无符号数

```c
// sizeof 返回 size_t（无符号类型）
if (sizeof(int) - sizeof(long long) > 0)
    // 32位系统：sizeof(int)=4, sizeof(long long)=8
    // 4-8 在无符号运算下变成很大的正数，条件为true！
```

### 陷阱5：移位量越界

```c
int x = 1;
int y = x << 32;  // 未定义行为！移位量>=字长
```

### 陷阱6：浮点数相等比较

```c
double x = 0.1 + 0.2;
if (x == 0.3)  // false！浮点精度问题
    ...
// 正确做法：
if (fabs(x - 0.3) < 1e-10)
    ...
```

## 十二、原码、反码与补码对比

| 表示方法 | 正数 | 负数编码 | 零的表示 | 加法复杂度 |
|---------|------|---------|---------|-----------|
| 原码 | 正常 | 符号位=1，其余不变 | +0/-0两种 | 需判断符号位 |
| 反码 | 正常 | 各位取反 | +0/-0两种 | 需循环进位修正 |
| 补码 | 正常 | 取反加一 | 唯一表示 | 统一加法电路 |

### 补码求负数的两种方法

**方法1：取反加一**
```
-5（8位）：
 5  = 0000 0101
取反 = 1111 1010
加一 = 1111 1011  ✓

验证：-128+64+32+16+8+2+1 = -128+123 = -5 ✓
```

**方法2：找最低位1，左边各位取反**
```
-5：0000 0101
最低位1在第0位，左边各位取反：
结果：1111 1011  ✓（与方法1相同）
```
