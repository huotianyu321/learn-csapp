## 4.1 Y86-64 指令集架构（ISA）

### 基本概念
- **ISA（Instruction Set Architecture）**：程序员可见的处理器状态和指令集的抽象定义，是软硬件之间的接口。
- Y86-64 是书中为教学目的设计的简化版 x86-64 ISA。

### 程序员可见状态
| 状态 | 说明 |
|------|------|
| 寄存器文件 | 15个64位通用寄存器（`%rax` ~ `%r14`，无 `%r15`） |
| 条件码 | `ZF`（零）、`SF`（符号）、`OF`（溢出） |
| 程序计数器 PC | 存放下一条指令地址 |
| 内存 | 字节寻址的虚拟内存 |
| 状态码 Stat | 反映程序执行状态（AOK/HLT/ADR/INS） |

### Y86-64 指令类型

#### 指令编码基础
- 每条指令由 **1字节指令码（icode:ifun）** + 可选字段（寄存器字节、立即数/地址）组成。
- `icode`：指令类别（高4位）；`ifun`：指令功能变体（低4位，如跳转条件）。
- 寄存器字节：高4位 `rA`，低4位 `rB`；`0xF` 表示无寄存器（`RNONE`）。
- 立即数/地址均以**小端序（little-endian）**存储，占8字节。

#### 寄存器编号对照
| 编号 | 寄存器 | 编号 | 寄存器 | 编号 | 寄存器 |
|------|--------|------|--------|------|--------|
| `0` | `%rax` | `5` | `%rbp` | `A` | `%r10` |
| `1` | `%rcx` | `6` | `%rsi` | `B` | `%r11` |
| `2` | `%rdx` | `7` | `%rdi` | `C` | `%r12` |
| `3` | `%rbx` | `8` | `%r8`  | `D` | `%r13` |
| `4` | `%rsp` | `9` | `%r9`  | `E` | `%r14` |
| `F` | 无寄存器（RNONE） | | | | |

#### 一、数据传送指令

**① `rrmovq rA, rB`** — 寄存器→寄存器，长度 **2 字节**

```
指令码   寄存器字节
 20       rA rB
```

示例：`rrmovq %rax, %rbx`（将 %rax 传送到 %rbx）
```
20 03
```
> `20`= icode:ifun（2:0），`03`= rA:rB（%rax=0, %rbx=3）

---

**② `irmovq V, rB`** — 立即数→寄存器，长度 **10 字节**

```
指令码   寄存器字节    立即数V（8字节，小端序）
 30       F  rB      v0 v1 v2 v3 v4 v5 v6 v7
```

示例：`irmovq $0x100, %rax`（将立即数 256 传送到 %rax）
```
30 F0 00 01 00 00 00 00 00 00
```
> `30`= icode:ifun（3:0），`F0`= rA:rB（RNONE=F, %rax=0），`00 01 00...00`= 0x100 的小端序

---

**③ `rmmovq rA, D(rB)`** — 寄存器→内存，长度 **10 字节**

```
指令码   寄存器字节    偏移量D（8字节，小端序）
 40       rA rB      d0 d1 d2 d3 d4 d5 d6 d7
```

示例：`rmmovq %rax, 8(%rsp)`（将 %rax 存入 %rsp+8 的内存）
```
40 04 08 00 00 00 00 00 00 00
```
> `40`= icode:ifun（4:0），`04`= rA:rB（%rax=0, %rsp=4），`08 00...00`= 偏移 8 的小端序

---

**④ `mrmovq D(rB), rA`** — 内存→寄存器，长度 **10 字节**

```
指令码   寄存器字节    偏移量D（8字节，小端序）
 50       rA rB      d0 d1 d2 d3 d4 d5 d6 d7
```

示例：`mrmovq 8(%rsp), %rax`（从 %rsp+8 的内存读入 %rax）
```
50 04 08 00 00 00 00 00 00 00
```
> `50`= icode:ifun（5:0），`04`= rA:rB（%rax=0, %rsp=4），`08 00...00`= 偏移 8 的小端序

---

#### 二、算术/逻辑指令（OPq）

**`OPq rA, rB`** — 寄存器运算，长度 **2 字节**，**更新条件码**

```
指令码   寄存器字节
icode:ifun  rA rB
```

| 指令 | icode:ifun | 示例（`%rax`, `%rbx`） | 编码 |
|------|-----------|----------------------|------|
| `addq` | `60` | `addq %rax, %rbx` | `60 03` |
| `subq` | `61` | `subq %rax, %rbx` | `61 03` |
| `andq` | `62` | `andq %rax, %rbx` | `62 03` |
| `xorq` | `63` | `xorq %rax, %rbx` | `63 03` |

> 寄存器字节 `03` = rA:%rax(0), rB:%rbx(3)；计算结果写回 rB，更新 ZF/SF/OF。

---

#### 三、跳转指令（jXX）

**`jXX Dest`** — 跳转到目标地址，长度 **9 字节**

```
指令码    目标地址（8字节，小端序）
icode:ifun  d0 d1 d2 d3 d4 d5 d6 d7
```

| 指令 | icode:ifun | 跳转条件 | 示例（跳转到地址 `0x400`） | 编码 |
|------|-----------|---------|--------------------------|------|
| `jmp` | `70` | 无条件 | `jmp 0x400` | `70 00 04 00 00 00 00 00 00` |
| `jle` | `71` | `ZF=1` 或 `SF≠OF` | `jle 0x400` | `71 00 04 00 00 00 00 00 00` |
| `jl`  | `72` | `SF≠OF` | `jl  0x400` | `72 00 04 00 00 00 00 00 00` |
| `je`  | `73` | `ZF=1` | `je  0x400` | `73 00 04 00 00 00 00 00 00` |
| `jne` | `74` | `ZF=0` | `jne 0x400` | `74 00 04 00 00 00 00 00 00` |
| `jge` | `75` | `SF=OF` | `jge 0x400` | `75 00 04 00 00 00 00 00 00` |
| `jg`  | `76` | `ZF=0` 且 `SF=OF` | `jg  0x400` | `76 00 04 00 00 00 00 00 00` |

> 地址 `0x400` 的小端序为 `00 04 00 00 00 00 00 00`。

---

#### 四、条件传送指令（cmovXX）

**`cmovXX rA, rB`** — 条件成立时传送，长度 **2 字节**，**不更新条件码**

| 指令 | icode:ifun | 条件 | 示例（`%rax→%rbx`） | 编码 |
|------|-----------|------|---------------------|------|
| `cmovle` | `21` | `ZF=1` 或 `SF≠OF` | `cmovle %rax, %rbx` | `21 03` |
| `cmovl`  | `22` | `SF≠OF` | `cmovl  %rax, %rbx` | `22 03` |
| `cmove`  | `23` | `ZF=1` | `cmove  %rax, %rbx` | `23 03` |
| `cmovne` | `24` | `ZF=0` | `cmovne %rax, %rbx` | `24 03` |
| `cmovge` | `25` | `SF=OF` | `cmovge %rax, %rbx` | `25 03` |
| `cmovg`  | `26` | `ZF=0` 且 `SF=OF` | `cmovg  %rax, %rbx` | `26 03` |

> 与 `rrmovq`（`20`）共享 icode=2，仅 ifun 不同；ifun=0 为无条件传送（即 `rrmovq`）。

---

#### 五、过程调用指令

**⑤ `call Dest`** — 调用，长度 **9 字节**

```
指令码    目标地址（8字节，小端序）
 80       d0 d1 d2 d3 d4 d5 d6 d7
```

示例：`call 0x400`（调用地址 0x400 处的函数）
```
80 00 04 00 00 00 00 00 00
```

---

**⑥ `ret`** — 返回，长度 **1 字节**

```
90
```

---

#### 六、栈操作指令

**⑦ `pushq rA`** — 压栈，长度 **2 字节**

```
指令码   寄存器字节
 A0       rA  F
```

示例：`pushq %rax`（压入 %rax）
```
A0 0F
```
> `A0`= icode:ifun（A:0），`0F`= rA:rB（%rax=0, RNONE=F）

---

**⑧ `popq rA`** — 出栈，长度 **2 字节**

```
指令码   寄存器字节
 B0       rA  F
```

示例：`popq %rax`（弹出栈顶到 %rax）
```
B0 0F
```
> `B0`= icode:ifun（B:0），`0F`= rA:rB（%rax=0, RNONE=F）

---

#### 七、停止指令

**`halt`** — 停止，长度 **1 字节**

```
00
```

---

#### 指令编码汇总表

| 指令 | icode | ifun | 附加字段 | 总长度 |
|------|-------|------|---------|--------|
| `halt` | `0` | `0` | — | 1 B |
| `nop` | `1` | `0` | — | 1 B |
| `rrmovq rA, rB` | `2` | `0` | 寄存器字节 | 2 B |
| `cmovXX rA, rB` | `2` | `1~6` | 寄存器字节 | 2 B |
| `irmovq V, rB` | `3` | `0` | 寄存器字节 + 8字节立即数 | 10 B |
| `rmmovq rA, D(rB)` | `4` | `0` | 寄存器字节 + 8字节偏移 | 10 B |
| `mrmovq D(rB), rA` | `5` | `0` | 寄存器字节 + 8字节偏移 | 10 B |
| `addq/subq/andq/xorq` | `6` | `0~3` | 寄存器字节 | 2 B |
| `jXX Dest` | `7` | `0~6` | 8字节目标地址 | 9 B |
| `call Dest` | `8` | `0` | 8字节目标地址 | 9 B |
| `ret` | `9` | `0` | — | 1 B |
| `pushq rA` | `A` | `0` | 寄存器字节 | 2 B |
| `popq rA` | `B` | `0` | 寄存器字节 | 2 B |

## 4.2 逻辑设计和硬件控制语言 HCL

### 基础逻辑门
- `AND`、`OR`、`NOT` 构成所有数字逻辑电路。
- **组合电路**：输出仅由当前输入决定，无记忆功能。
- **时序电路**：带有存储单元（寄存器/内存），输出依赖历史状态。

### HCL 表达式
```
# 布尔表达式
bool eq = (a == b);

# 字级多路选择（case表达式）
word out = [
    s : a;   # 若 s 为真，选 a
    1 : b;   # 否则选 b
];
```

### 关键硬件单元
| 单元 | 作用 |
|------|------|
| 算术逻辑单元 ALU | 执行算术和逻辑运算 |
| 寄存器文件 | 读/写通用寄存器，有两个读端口和一个写端口 |
| 指令内存 | 读取指令字节 |
| 数据内存 | 读写数据 |
| PC（程序计数器）寄存器 | 存储当前指令地址 |

## 4.3 顺序实现（SEQ）

### 指令执行的六个阶段

```
取指（Fetch）→ 译码（Decode）→ 执行（Execute）→ 访存（Memory）→ 写回（Write Back）→ 更新PC
```

| 阶段 | 工作内容 |
|------|---------|
| **取指** | 从PC所指内存读取指令字节，解析 icode、ifun，读取寄存器编号、立即数，计算 valP（顺序下一条指令地址） |
| **译码** | 从寄存器文件读取操作数 valA、valB |
| **执行** | ALU 计算：运算结果 valE 或地址计算；设置/使用条件码 |
| **访存** | 读/写数据内存，得到 valM |
| **写回** | 将 valE 或 valM 写入目的寄存器 |
| **更新PC** | 将 PC 设置为下一条指令地址 |

### 各指令六阶段详细示例

> 符号说明：`M₁[x]` 表示从地址 x 读取1字节；`M₈[x]` 表示读/写8字节；`R[r]` 表示寄存器 r 的值；`valC` 为指令中的立即数或偏移量；`valP` 为下一条指令的PC。

---

#### ① `OPq rA, rB`（整数运算：add/sub/and/xor）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `rA:rB ← M₁[PC+1]` <br> `valP ← PC+2` |
| **译码** | `valA ← R[rA]` <br> `valB ← R[rB]` |
| **执行** | `valE ← valB OP valA` <br> 设置条件码 CC |
| **访存** | 无 |
| **写回** | `R[rB] ← valE` |
| **更新PC** | `PC ← valP` |

> `OP` 根据 `ifun` 确定为加、减、与、异或之一；结果写回 rB，**更新 ZF/SF/OF**。

---

#### ② `rrmovq rA, rB`（寄存器到寄存器传送）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `rA:rB ← M₁[PC+1]` <br> `valP ← PC+2` |
| **译码** | `valA ← R[rA]` |
| **执行** | `valE ← 0 + valA`（ALU 计算，ifun=0） |
| **访存** | 无 |
| **写回** | `R[rB] ← valE` |
| **更新PC** | `PC ← valP` |

> 利用 ALU 做 `0 + valA` 来传送数据，**不更新条件码**（条件传送 `cmovXX` 同框架，执行阶段增加条件判断）。

---

#### ③ `irmovq V, rB`（立即数到寄存器传送）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `rA:rB ← M₁[PC+1]`（rA = F，即 RNONE） <br> `valC ← M₈[PC+2]` <br> `valP ← PC+10` |
| **译码** | 无（源操作数为立即数，不读寄存器） |
| **执行** | `valE ← 0 + valC` |
| **访存** | 无 |
| **写回** | `R[rB] ← valE` |
| **更新PC** | `PC ← valP` |

> 立即数 V 编码为指令中的 8 字节常数 `valC`，指令总长 **10 字节**，**不更新条件码**。

---

#### ④ `rmmovq rA, D(rB)`（寄存器到内存传送）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `rA:rB ← M₁[PC+1]` <br> `valC ← M₈[PC+2]`（偏移量 D） <br> `valP ← PC+10` |
| **译码** | `valA ← R[rA]` <br> `valB ← R[rB]` |
| **执行** | `valE ← valB + valC`（计算有效地址） |
| **访存** | `M₈[valE] ← valA`（将 rA 写入内存） |
| **写回** | 无 |
| **更新PC** | `PC ← valP` |

> 有效地址 = `rB + D`；源数据 `valA` 写入内存，**不写寄存器，不更新条件码**。

---

#### ⑤ `mrmovq D(rB), rA`（内存到寄存器传送）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `rA:rB ← M₁[PC+1]` <br> `valC ← M₈[PC+2]`（偏移量 D） <br> `valP ← PC+10` |
| **译码** | `valB ← R[rB]`（用于地址计算） |
| **执行** | `valE ← valB + valC`（计算有效地址） |
| **访存** | `valM ← M₈[valE]`（从内存读取数据） |
| **写回** | `R[rA] ← valM` |
| **更新PC** | `PC ← valP` |

> 与 `rmmovq` 互为逆操作；**流水线中此指令后紧跟使用 rA 的指令会产生加载-使用冒险**，须暂停1周期。

---

#### ⑥ `pushq rA`（压栈）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `rA:rB ← M₁[PC+1]`（rB = F） <br> `valP ← PC+2` |
| **译码** | `valA ← R[rA]` <br> `valB ← R[%rsp]` |
| **执行** | `valE ← valB + (-8)`（栈指针减8） |
| **访存** | `M₈[valE] ← valA`（写入数据到新栈顶） |
| **写回** | `R[%rsp] ← valE`（更新栈指针） |
| **更新PC** | `PC ← valP` |

---

#### ⑦ `popq rA`（出栈）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `rA:rB ← M₁[PC+1]`（rB = F） <br> `valP ← PC+2` |
| **译码** | `valA ← R[%rsp]` <br> `valB ← R[%rsp]` |
| **执行** | `valE ← valB + 8`（栈指针加8） |
| **访存** | `valM ← M₈[valA]`（从旧栈顶读数据） |
| **写回** | `R[%rsp] ← valE` <br> `R[rA] ← valM` |
| **更新PC** | `PC ← valP` |

> **注意**：rA 与 %rsp 可能相同（如 `popq %rsp`），写回阶段 **valM 优先于 valE** 写入 rA。

---

#### ⑧ `call Dest`（过程调用）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `valC ← M₈[PC+1]`（目标地址 Dest） <br> `valP ← PC+9` |
| **译码** | `valB ← R[%rsp]` |
| **执行** | `valE ← valB + (-8)` |
| **访存** | `M₈[valE] ← valP`（压入返回地址） |
| **写回** | `R[%rsp] ← valE` |
| **更新PC** | `PC ← valC`（跳转到目标地址） |

---

#### ⑨ `ret`（过程返回）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `valP ← PC+1` |
| **译码** | `valA ← R[%rsp]` <br> `valB ← R[%rsp]` |
| **执行** | `valE ← valB + 8` |
| **访存** | `valM ← M₈[valA]`（读取返回地址） |
| **写回** | `R[%rsp] ← valE` |
| **更新PC** | `PC ← valM`（跳转到返回地址） |

> 流水线中 `ret` 的返回地址需到访存阶段才知道，**须暂停3个周期**（插入3个气泡）等待。

---

#### ⑩ `jXX Dest`（条件/无条件跳转）

| 阶段 | 操作 |
|------|------|
| **取指** | `icode:ifun ← M₁[PC]` <br> `valC ← M₈[PC+1]`（跳转目标） <br> `valP ← PC+9` |
| **译码** | 无 |
| **执行** | `Cnd ← Cond(CC, ifun)`（根据条件码和 ifun 计算跳转条件） |
| **访存** | 无 |
| **写回** | 无 |
| **更新PC** | `PC ← Cnd ? valC : valP` |

> `jmp`（ifun=0）无条件跳转；其余根据 CC 决定。流水线预测**不跳转（valP）**，若实际跳转则取消后续2条已取指指令。

---

### 各指令阶段对比汇总

| 指令 | 指令长度 | 更新CC | 读内存 | 写内存 | 写寄存器 |
|------|---------|--------|--------|--------|---------|
| `OPq rA, rB` | 2 B | ✅ | — | — | rB |
| `rrmovq rA, rB` | 2 B | — | — | — | rB |
| `irmovq V, rB` | 10 B | — | — | — | rB |
| `rmmovq rA, D(rB)` | 10 B | — | — | ✅ | — |
| `mrmovq D(rB), rA` | 10 B | — | ✅ | — | rA |
| `pushq rA` | 2 B | — | — | ✅ | %rsp |
| `popq rA` | 2 B | — | ✅ | — | rA, %rsp |
| `call Dest` | 9 B | — | — | ✅ | %rsp |
| `ret` | 1 B | — | ✅ | — | %rsp |
| `jXX Dest` | 9 B | — | — | — | — |

### SEQ 的缺点
- 每个时钟周期只执行一条指令，时钟频率受制于最长指令路径（关键路径）。
- 硬件资源利用率低，不同阶段的部件在同一周期内大量闲置。

## 4.4 流水线通用原理

### 流水线基本思想
- 将指令执行划分为多个**独立阶段**，每个阶段由不同硬件完成。
- 多条指令同时处于不同阶段——**指令级并行**。
- 理想加速比 ≈ 流水线级数，吞吐量大幅提升。

### 流水线局限
| 问题 | 说明 |
|------|------|
| 不一致的阶段延迟 | 各阶段耗时不同，整体受**最慢阶段**限制 |
| 流水线寄存器开销 | 级间寄存器增加额外延迟 |
| 流水线冒险 | 相关性导致无法继续流水（见下节） |

---

## 4.5 流水线实现（PIPE）

### SEQ+ 到 PIPE 的改造
1. **SEQ+**：将"更新PC"移到流水线起始，使PC计算先行。
2. **PIPE**：在各阶段之间插入**流水线寄存器**（F/D/E/M/W），保存中间值。

### 五级流水线寄存器
```
PC → [F寄存器] → D → [D寄存器] → E → [E寄存器] → M → [M寄存器] → W
```
- `F`：保存预测的PC值
- `D`：保存取指结果（icode、ifun、rA、rB、valC、valP）
- `E`：保存译码结果（valA、valB、目的寄存器、ALU操作）
- `M`：保存执行结果（valE、条件码、跳转目标）
- `W`：保存访存结果（valE、valM、写回目的）

### 三类流水线冒险

#### 1. 数据冒险（Data Hazard）
后续指令需要前面指令尚未写回的结果。

**解决方案：**

| 方法 | 说明 |
|------|------|
| **转发（Forwarding / Bypassing）** | 将执行/访存/写回阶段的中间结果直接送给需要的阶段，无需等待写回 |
| **暂停（Stalling）** | 向流水线插入气泡（bubble），延迟后续指令 |
| **加载-使用冒险** | `mrmovq` 后紧跟使用该寄存器的指令，必须暂停1个周期（转发不够） |

**转发来源优先级（valA/valB 选择）：**
```
e_valE > m_valE > W_valE > m_valM > W_valM > 寄存器文件
```

#### 2. 控制冒险（Control Hazard）
跳转指令、`ret` 指令导致正确的下一条 PC 不确定。

**解决方案：**

| 情况 | 处理方式 |
|------|---------|
| 条件跳转 | **预测跳转不发生（predict not-taken）**；若实际跳转，取消已取指的指令（插入bubble） |
| `ret` | 暂停3个周期直到 `ret` 写回PC；或使用返回地址预测 |

#### 3. 结构冒险（Structural Hazard）
多条指令竞争同一硬件资源（Y86-64中较少，已通过分离指令/数据内存避免）。

### 异常处理
- 状态码（Stat）随指令在流水线中传播。
- 当异常指令到达写回阶段时才对外生效，保证精确异常。
- 异常指令之后进入流水线的指令用气泡替代，禁止修改状态。

## 4.6 关键信号与控制逻辑

### 流水线控制信号
| 信号 | 作用 |
|------|------|
| `stall`（暂停） | 保持流水线寄存器内容不变（freeze） |
| `bubble`（气泡） | 将流水线寄存器重置为 nop 指令等效状态 |
| 正常 | 在时钟上升沿加载新值 |

### 需要暂停/气泡的典型场景

| 场景 | F寄存器 | D寄存器 | E寄存器 |
|------|--------|--------|--------|
| 加载-使用冒险 | stall | stall | bubble |
| ret 在流水线中 | stall | bubble | 正常 |
| 误预测跳转 | 正常 | bubble | bubble |

## 4.7 性能分析

### CPI（每指令周期数）
$$
\text{CPI} = 1.0 + \text{lp} + \text{mp} + \text{rp}
$$

| 惩罚项 | 含义 |
|--------|------|
| `lp`（load penalty） | 加载-使用冒险引起的平均暂停周期 |
| `mp`（misprediction penalty） | 跳转预测错误的平均惩罚 |
| `rp`（return penalty） | ret指令的平均暂停周期 |

- 理想CPI = 1.0（无冒险）
- 实际 CPI 通常在 1.0~1.5 之间（取决于程序特征）

### 重要结论
1. **ISA 是抽象层**：隔离软件与硬件实现细节，同一 ISA 可有多种微架构实现。
2. **流水线是提升吞吐量的核心手段**，但不降低单条指令延迟。
3. **数据冒险首选转发**，转发无法解决的才用暂停（如加载-使用）。
4. **控制冒险代价高**，现代处理器采用复杂分支预测器（本书用简单预测不跳转）。
5. **异常处理需精确**：确保异常之前的指令完成，之后的指令不生效。
6. **硬件描述语言（HCL）** 将设计意图与门级实现分离，便于验证和综合。
